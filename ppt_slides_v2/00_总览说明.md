# PPT 制作总览 — 多源遥感冲突损害贝叶斯融合

## 本PPT在讲什么

以**同学的两传感器完整方案为主线**, 用10页PPT讲完一个完整故事:

> "Sentinel-2和VIIRS两颗卫星看同一个战区, 给出了完全不同的损毁判断。用贝叶斯层次模型把它们的信息整合起来, 结果发现——传感器不是在互相印证, 而是在互相否定。但这恰恰是融合的价值所在: 不是找正确答案, 而是诚实说'我不确定'。"

## 10页逻辑线

```
(1) 为什么需要多传感器? → 卫星各有盲区
(2) 我们用了什么数据? → S2光谱 + VIIRS灯光 + UNOSAT标注
(3) S2如何从卫星影像变成损毁评分? → 5步处理流程
(4) VIIRS如何从灯光变成损毁评分? → 4步处理流程
(5) 如何融合两个传感器? → 贝叶斯层次模型(MCMC)
(6) 融合结果是什么? → 两传感器互相矛盾, 融合退化为"不确定"
(7) 如何量化"矛盾"? → KLD信息增益分析, 46.8%抑制率
(8) 换个融合方法会怎样? → D-S vs Bayesian对比
(9) 有办法验证到底准不准吗? → UNOSAT外部参考
(10) 如果加更多传感器呢? → 四传感器拓展 + 两方案互补
```

## 每一页的核心消息

| 页 | 核心消息(一句话) |
|----|-----------------|
| 1 | S2测结构变化, VIIRS测功能丧失, UNOSAT人工标注——三个源看到损毁的不同维度 |
| 2 | 同一城市同一时期, S2说"这里变了", VIIRS说"那里灯灭了"——两幅图完全不同 |
| 3 | 从13波段卫星影像→3个遥感指数→PCA降维→方向感知评分→GMM分堆 = 损毁概率输出 |
| 4 | NTL损失率归一化→Beta分布CDF映射——VIIRS告诉你灯光有没有消失 |
| 5 | 贝叶斯三层模型: 隐变量z_i + 传感器精度κ_k + MCMC推断 = 诚实的不确定性 |
| 6 | 核心发现: 两传感器p值几乎零相关, 融合后验退化为均匀分布(p≈0.155) |
| 7 | KLD分析: 融合没有增加信息量! 因为两传感器互斥, 信息被互相抵消 |
| 8 | D-S丢弃冲突→过于保守(Bel→0); 贝叶斯量化冲突→收敛至先验(p→0.5) |
| 9 | UNOSAT验证: S2 AUC≈0.67, VIIRS≈0.52——装修正的"损毁"是建筑结构损毁 |
|10 | 加SAR+InSAR→四个传感器|ρ|<0.11→两方案互补: MCMC+四传感器=完整方案 |

## 文件夹内容

每个 page_XX 文件夹包含:
- 1-2张图片(.png, 论文级质量)
- 说明.txt: 详细解释本页在讲什么、每张图怎么看、如何排版

## 制作规范

- 比例: 16:9
- 标题: 28pt Bold
- 正文: 16-18pt
- 图注: 12pt
- 默认使用白底+传感器固定配色(VIIRS=蓝, S2=绿, SAR=橙, InSAR=紫)

## 使用方式

1. 打开PowerPoint, 新建16:9空白演示文稿
2. 从第1页到第10页, 每页按对应文件夹中的图片和说明制作
3. 每页放入对应图片→添加标题→按说明添加文字
4. 第2-10页的文字控制在每页20字以内(当有两张图时)
