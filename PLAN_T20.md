# T20 多传感器贝叶斯不确定性融合 — 冲突损害层次模型

## 作业要求摘要

T20 原本要求融合 **4 种遥感数据源**，按统计方法链完成：
1. 每传感器独立输出损害概率 p_k ± σ²_k
2. 贝叶斯层次模型 (Level1: Dᵢ~Bernoulli(pᵢ), Level2: y_ik|Dᵢ~Beta-Binomial, Level3: pᵢ~Beta(a,b))
3. MCMC 后验采样 → 95%CI
4. KLD 信息增益评估各传感器价值
5. Dempster-Shafer 证据理论融合 (Bel ≤ P ≤ Pl)
6. LOOCV → ROC/AUC
7. 贝叶斯融合 vs 简单投票 vs D-S → AUC 比较

**本方案先从 2 种传感器做起，框架预留扩展到 4 种的能力。**

## 总体方法框架（适配 2 传感器版）

```
┌─────────────────────────────────────────────────────────────┐
│                    阶段 1: 单传感器损害概率                    │
├─────────────────────────────────────────────────────────────┤
│  VIIRS NTL ──→ 灯光变化量 ΔNTL ──→ Logistic/阈值 ──→ p₁,σ²₁ │
│  Sentinel-2 ──→ 多光谱指数变化 ──→ PCA/Hotelling T² ──→ p₂,σ²₂│
└──────────────────────┬──────────────────────────────────────┘
                       ▼
┌─────────────────────────────────────────────────────────────┐
│                    阶段 2: 贝叶斯层次融合                      │
├─────────────────────────────────────────────────────────────┤
│  层次模型:                                                    │
│    Level 1: Dᵢ ~ Bernoulli(pᵢ)     ← 像素 i 的真实损害状态    │
│    Level 2: yᵢₖ|Dᵢ ~ Normal(μₖ(Dᵢ), σ²ₖ) ← 传感器 k 的观测  │
│    Level 3: pᵢ ~ Beta(α, β)         ← 损害概率的先验          │
│  MCMC (PyMC) → 后验 pᵢ|y → 95% CI → 损害概率图 + 不确定性图    │
└──────────────────────┬──────────────────────────────────────┘
                       ▼
┌─────────────────────────────────────────────────────────────┐
│                    阶段 3: 证据理论融合                        │
├─────────────────────────────────────────────────────────────┤
│  Dempster-Shafer 组合规则:                                    │
│    Bel(损害) ≤ P(损害) ≤ Pl(损害)                             │
│    与贝叶斯融合结果对比一致性                                   │
└──────────────────────┬──────────────────────────────────────┘
                       ▼
┌─────────────────────────────────────────────────────────────┐
│                    阶段 4: 模型评估与对比                       │
├─────────────────────────────────────────────────────────────┤
│  LOOCV → ROC/AUC                                              │
│  三种融合方法对比: 贝叶斯层次 vs 简单投票 vs D-S                 │
│  KLD 信息增益评估每个传感器的贡献                               │
└─────────────────────────────────────────────────────────────┘
```

## 详细实施步骤

### Step 1: 数据预处理与对齐 (预计 1-2 天)

**1.1 VIIRS 处理**
- 读取 HDF5 中 `DNB_BRDF-Corrected_NTL` 和 `Gap_Filled_DNB_BRDF-Corrected_NTL`
- 使用 Quality Flag 过滤低质量像素
- 计算战前-战后差异: ΔNTL = NTL_post - NTL_pre
- 归一化: 将 ΔNTL 映射到 [0,1] 作为初步损害分数
- 重采样/重投影到统一网格

**1.2 Sentinel-2 处理**
- 读取 10m 波段 (B2 Blue, B3 Green, B4 Red, B8 NIR)
- 计算多光谱指数:
  - NDVI = (NIR - Red) / (NIR + Red) — 植被变化
  - NBR = (NIR - SWIR2(B12)) / (NIR + SWIR2) — 火烧迹地
  - NDBI = (SWIR1(B11) - NIR) / (SWIR1 + NIR) — 建筑指数
  - NDWI = (Green - NIR) / (Green + NIR) — 水体变化
- 计算战前-战后差异: ΔNDVI, ΔNBR, ΔNDBI, ΔNDWI
- 使用 SCL 掩膜云/云影/水等无关类别

**1.3 空间对齐**
- VIIRS 地理坐标 (WGS84 lat/lon) → 重投影到 EPSG:32637
- Sentinel-2 保持 EPSG:32637
- 统一到共同网格 (建议 100m 或 200m，兼顾两种分辨率)
- 处理两景 Sentinel-2 tile (T37TDN + T37TCN) 的镶嵌

### Step 2: 单传感器损害概率估计 (预计 2-3 天)

**2.1 VIIRS → 损害概率 p₁**
- 方法 A: 高斯混合模型 (GMM) 对 ΔNTL 聚类 → "损害"类后验概率
- 方法 B: 基于阈值的 Logistic 回归 (用高质量/低质量区域标注训练)
- 输出: 每个像素的 p₁ 和方差 σ²₁ (Bootstrap 或 Beta 分布参数)
- 可视化: ΔNTL 空间分布图 + p₁ 概率图

**2.2 Sentinel-2 → 损害概率 p₂**
- 将 ΔNDVI, ΔNBR, ΔNDBI, ΔNDWI 组成 4 维特征向量
- PCA 降维 → PC1 作为综合损害指数
- Hotelling T² 检验: H₀: 战前=战后 (多元均值检验)
- Benjamini-Hochberg FDR 多重比较校正
- 每个显著变化像素 → p₂ = 1 - FDR-adjusted p-value
- 方差 σ²₂ 通过 Bootstrap 重采样估计
- 可视化: PC1 损害指数图 + p₂ 显著性图

### Step 3: 贝叶斯层次模型融合 (预计 2-3 天)

**3.1 模型构建 (PyMC)**
```python
# Level 3: 全局先验
α ~ HalfNormal(5)
β ~ HalfNormal(5)

# Level 1: 像素级损害概率
p_i ~ Beta(α, β)  for i in 1..N

# Level 2: 传感器观测模型
# VIIRS
y_i1 | p_i ~ Normal(μ_1 + γ_1 * logit(p_i), σ²_1)
# Sentinel-2  
y_i2 | p_i ~ Normal(μ_2 + γ_2 * logit(p_i), σ²_2)
```

**3.2 MCMC 采样**
- NUTS (No-U-Turn Sampler), 4 chains, 2000 warmup + 2000 samples
- 收敛诊断: R-hat < 1.01, ESS > 400
- 后验分布: p_i|y 的中位数 + 95% HDI

**3.3 输出**
- 融合损害概率图 (后验均值)
- 不确定性图 (后验标准差 / HDI 宽度)
- Trace plot, posterior density plot

### Step 4: Dempster-Shafer 证据理论融合 (预计 1-2 天)

**4.1 构建 mass 函数**
- Frame of discernment: Θ = {damaged, not_damaged}
- 从 p₁, p₂ 构建基本概率分配 (BPA):
  - m₁({damaged}) = p₁, m₁({not_damaged}) = 1-p₁-ε, m₁(Θ) = ε (从 σ²₁ 推导)
  - m₂ 同理

**4.2 D-S 组合规则**
- m₁₂ = m₁ ⊕ m₂ (Dempster 组合)
- 输出: Bel(损害), Pl(损害), [Bel, Pl] 区间
- 区间宽度 = Pl - Bel = 不确定性度量

### Step 5: 模型评估与对比 (预计 1-2 天)

**5.1 KLD 信息增益**
- KL(p_fused || p_single) 量化每个传感器的边际贡献
- KL(p_fused || p_uniform) 量化融合方案的信息增益

**5.2 LOOCV**
- 留一交叉验证 (像素级)
- ROC 曲线 + AUC 值

**5.3 三种方法对比**
| 方法 | AUC | 不确定性量化 | 计算复杂度 |
|------|-----|-------------|-----------|
| 贝叶斯层次融合 | ? | 完整后验 | 最高 |
| 简单投票 (平均 p) | ? | 无 | 最低 |
| D-S 证据理论 | ? | Bel/Pl 区间 | 中等 |

**5.4 可视化对比**
- 三种方法的损害概率图并列展示
- 不确定性/置信区间对比
- 散点图: 贝叶斯 p vs 投票 p vs D-S Bel

- 引言 (冲突遥感监测背景)
- 数据描述 (VIIRS + Sentinel-2)

- 结果 (各步骤输出图+表)

- 结论

## 关键难点与注意事项

1. **空间分辨率差异大**: VIIRS 500m vs Sentinel-2 10m → 上采样统一网格会丢失信息，考虑在融合模型中显式建模分辨率差异
2. **云覆盖**: Sentinel-2 光学影像受云影响，必须用 SCL + Cloud Mask 严格过滤
3. **无地面真值**: T20 方案中应讨论使用 OSM 建筑轮廓或手动标注作为验证标签，或依赖"软验证"(如两种传感器一致区域置信度高)
4. **2 传感器局限**: KLD 信息增益分析只有 2 个自由度, D-S 融合没有多数投票机制 → 在讨论中明确说明扩展到 4 传感器的路径
5. **计算量**: MCMC 在全像素网格上运行不现实 → 策略:
   - 降采样到 ~200m 网格
   - 或采样代表性像素子集建模型 + 后验预测插值到全图
