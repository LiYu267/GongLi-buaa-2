# PPT 制作说明 — 双方案对比: 多传感器遥感损毁评估

## 整体逻辑

本PPT以**"两套方案, 同一个问题"**为主线, 展示马里乌波尔冲突损害的多源遥感统计融合评估。10页按以下逻辑展开:

```
提出问题(1) → 方案总览(2) → 数据(3)
  → 方案A详解: 方法(4) → 结果(5)
  → 方案B详解: 方法(6) → 结果(7)
  → 核心对比(8) → 深层分析(9) → 结论(10)
```

**核心叙事**: 
- 方案A (两传感器 S2+VIIRS) 方法更完整 — 有MCMC、UNOSAT地面验证、KLD信息论分析
- 方案B (四传感器 +SAR+InSAR) 传感器更全面 — 但SAR/InSAR处理较简易, 缺乏地面真值验证
- 两方案共同发现: **不同传感器测量的是损毁的不同物理维度, 信号近乎正交(|ρ|<0.1), 验证了多源融合的必要性**

## 核心叙事时间线

```
1. 问题 → 2. 方案 → 3. 数据
  → 4-5: 方案A深挖(方法→结果, 发现"信号冲突")
  → 6-7: 方案B深挖(方法→结果, 发现"传感器正交")
  → 8: 同框对比 ← 全PPT高潮
  → 9: 深层原因(为什么冲突/正交? 信息论+KLD+D-S揭示)
  → 10: 整合路径(两方案互补, 未来方向)
```

**一句话总结10页**: 用两套方案共同证明"多传感器遥感损毁评估, 核心挑战不是融合技术, 而是传感器测量的是不同物理维度 — 所以融合不是为了找到'正确答案', 而是为了诚实量化'我们有多不确定'。"
